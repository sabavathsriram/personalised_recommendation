CREATION OF MAVEN WEB JAVA PROJECT

Step 1: Open Eclipse
   ‚îî‚îÄ‚îÄ 1.1 Launch Eclipse IDE.
   ‚îî‚îÄ‚îÄ 1.2 Select or create a workspace.

Step 2: Create a New Maven Project
   ‚îî‚îÄ‚îÄ 2.1 File -> New -> Project...
       ‚îî‚îÄ‚îÄ 2.1.1 Expand "Maven"
       ‚îî‚îÄ‚îÄ 2.1.2 Select "Maven Project" and click "Next"

Step 3: Choose Maven Archetype
   ‚îî‚îÄ‚îÄ 3.1 Select an archetype:
           'org.apache.maven.archetypes' -> 'maven-archetype-webapp' (version 1.4)
   ‚îî‚îÄ‚îÄ 3.2 Click "Next"

Step 4: Configure the Maven Project
   ‚îî‚îÄ‚îÄ 4.1 Group Id: e.g., com.example
   ‚îî‚îÄ‚îÄ 4.2 Artifact Id: e.g., my-web-app
   ‚îî‚îÄ‚îÄ 4.3 Click "Finish"

Step 5: Add Maven Dependencies
   ‚îî‚îÄ‚îÄ 5.1 Open "pom.xml" located in the root of your project (NOT inside target folder).
   ‚îî‚îÄ‚îÄ 5.2 Add required web dependencies (Servlet, JSP, etc.)
       ‚îî‚îÄ‚îÄ Go to browser -> open mvnrepository.com
       ‚îî‚îÄ‚îÄ Search for "Java Servlet API"
       ‚îî‚îÄ‚îÄ Select the latest version
       ‚îî‚îÄ‚îÄ Copy the dependency and add it inside <dependencies> in pom.xml

       Example:
       <dependency>
           <groupId>javax.servlet</groupId>
           <artifactId>javax.servlet-api</artifactId>
           <version>4.0.1</version>
           <scope>provided</scope>
       </dependency>

   ‚îî‚îÄ‚îÄ 6.1 Right-click project -> Maven -> Update Project (Force Update)
   ‚îî‚îÄ‚îÄ 6.2 Right-click project -> Properties -> Project Facets
           Enable:
           ‚Ä¢ Dynamic Web Module
           ‚Ä¢ Java
           ‚Ä¢ JavaScript (optional)
   ‚îî‚îÄ‚îÄ 6.3 Apply and click OK

   ‚îî‚îÄ‚îÄ 7.1 Window -> Show View -> Servers
   ‚îî‚îÄ‚îÄ 7.2 Click "Add New Server"
   ‚îî‚îÄ‚îÄ 7.3 Select "Tomcat v9.0 Server" -> Next
   ‚îî‚îÄ‚îÄ 7.4 Set server installation directory and ports if needed
   ‚îî‚îÄ‚îÄ 7.5 Click "Finish"

   ‚îî‚îÄ‚îÄ 8.1 Right-click on the PROJECT (not file)
       -> Run As -> Maven Clean
   ‚îî‚îÄ‚îÄ 8.2 Right-click on the PROJECT
       -> Run As -> Maven Install
   ‚îî‚îÄ‚îÄ 8.3 Right-click on the PROJECT

DOCKER

WORKING WITH DOCKER CLI COMMANDS (REDIS)


SECTION 1: DOCKER CLI COMMANDS WITH REDIS


Step 1: Pull the Redis Image
Command:
    docker pull redis

Step 2: Run a Redis Container
Command:
    docker run --name my-redis -d redis
What It Does:
    ‚Ä¢ Creates and starts a container named "my-redis" from the redis image.
    ‚Ä¢ The -d flag runs the container in the background.

Step 3: Check Running Containers
Command:
    docker ps
What It Does:
    ‚Ä¢ Lists all running containers.

Step 4: Access Redis Inside the Container
Command:
    docker exec -it my-redis redis-cli
What It Does:
    ‚Ä¢ Opens the Redis command-line tool (redis-cli) inside the running container.

Example Redis Commands:
    127.0.0.1:6379> SET name "Alice"
    OK
    127.0.0.1:6379> GET name
    "Alice"

Step 5: Stop the Redis Container
Command:
    docker stop my-redis
What It Does:
    ‚Ä¢ Stops the Redis container without deleting it.

Step 6: Restart the Redis Container
Command:
    docker start my-redis
What It Does:
    ‚Ä¢ Starts the previously stopped container.

Step 7: Remove the Redis Container
Command:
    docker rm my-redis
What It Does:
    ‚Ä¢ Deletes the container permanently.

Step 8: Remove the Redis Image
Command:
    docker rmi redis
What It Does:
    ‚Ä¢ Deletes the Redis image from your local system.



SECTION 2: WORKING WITH DOCKERFILE


A Dockerfile is a text file containing instructions to create a custom Docker image.

Step 1: Set Up Your Folder


1. Windows:
      ‚Ä¢ Create a folder: C:\DockerProjects\Redis
      ‚Ä¢ Open Git Bash and navigate to the folder:
            cd /c/DockerProjects/Redis

      ‚Ä¢ Create a folder:
            mkdir ~/DockerProjects/Redis
      ‚Ä¢ Navigate inside it:
            cd ~/DockerProjects/Redis


Step 2: Write the Dockerfile


1. Inside the folder, create a new file named:
       Dockerfile
   (no extension)

       FROM redis:latest
       CMD ["redis-server"]
